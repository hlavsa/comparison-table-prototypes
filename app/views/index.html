CAIT Comparison Table Prototypes

These options have gone through several iterations, according to the needs of the particular service we're working on.

Phase 1

In this phase, I took existing patterns and tables that others across government had already created. I used these things as a base for a series of potential options for our form.

Option 1 - Responsive Plain Table

This example is responsive but doesn't have the ability to expand content out on mobile.

Option 2 - Responsive Details Table

This option is responsive, and allows you to hide and expand content using the Design System details component

Option 3 - Responsive Accordion Table

This table is responsive, and allows you to hide and expand content using the Design System accordion component

Option 4 - Unresponsive Scroll Table

This table is responsive, and allows you to hide and expand content using the Design System accordion component

Phase 2

The Service Designer and I discussed the Phase 1 options. We decided that Option 1 and Option 4 were the most appropriate for our particular use case, but that we liked the benefits of Option 3. So, in this phase, we're making some adjustments to them to try and bring in the benefits that Option 3 offers.

We've also decided to rearrange the headers and table headers a little, and incorporate links to the respective options' pages.

Option 1 - Responsive Plain Table

In this iteration, we took inspiration from the Tab pattern on the Design System. We liked that accordions allowed you to switch between different groups of information, but found it didn't work on desktop. So, we're pulling across the 'Table of Contents' idea that is on the mobile version of the Tab component, and incorporating it into this table.

Option 4 - Unresponsive Scroll Table

In this iteration, we've adjusted the table headers to make them links, as including links is also a requirement of our table.
